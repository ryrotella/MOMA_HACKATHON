## What Was Built

### Project Scaffolded
- Next.js 16 (App Router) + TypeScript + Tailwind CSS + Zustand
- Dependencies: react-leaflet, leaflet, framer-motion, html-to-image, lottie-react, react-insta-stories
- `.env.local` with MoMA API token and Replicate API key

### Data Layer
- **57 curated artworks** in `src/data/artworks.json` with verified gallery/floor assignments
- All 57 artworks have **real MoMA image URLs** sourced from the MoMA GitHub CSV dataset (`MuseumofModernArt/collection`)
- Gallery coordinate mappings in `src/data/galleries.json` for floors 2, 4, and 5
- Zustand store (`src/store/useStore.ts`) with localStorage persistence for bookmarks, visit sessions, dwell time tracking

### SVG Floor Plans
- Custom SVGs for floors 2, 4, and 5 created from MoMA.org map screenshots
- Redesigned mid-session from 1200x700 (landscape with wasted space) to 780x580 (gallery-focused, no sculpture garden/amenity padding)
- Gallery labels, wing annotations, color-coded by collection era

### Pages

| Route | Description |
|-------|-------------|
| `/` | Homepage — MoMA branding, Start Exploring / View Wrapped CTAs, stats |
| `/map` | Interactive floor map with react-leaflet (CRS.Simple), artwork pins, gallery popups with thumbnails, floor switcher, 2D/3D isometric toggle |
| `/artwork/[id]` | Artwork detail — real MoMA images, metadata cards, tags, bookmark, dwell-time tracking, related works |
| `/wrapped` | 8-slide Spotify Wrapped-style story experience with framer-motion animations, shareable card download/share |
| `/api/moma` | Proxy route for MoMA API (for future use if token refreshed) |

### Key Features
- **Interactive map**: react-leaflet with SVG overlays, auto-fit to content on mobile, custom zoom controls
- **Isometric toggle**: CSS perspective transforms tilt map into 3D view, pins counter-rotate to stay upright
- **Artwork popups**: show real thumbnails, titles, artists, bookmark buttons
- **Wrapped stories**: welcome → stats → top artworks → Art DNA bars → soulmate artist → hidden gem → art personality → shareable card
- **Visit tracking**: session start/end, artwork views with timestamps, dwell time, floors/galleries visited
- **Bookmarking**: persisted in localStorage via Zustand
- **Shareable cards**: `html-to-image` generates PNG, Web Share API for native mobile share sheet
- **Bottom navigation**: Home / Map / Wrapped with active state indicators

### Mobile Optimizations
- SVGs cropped to gallery-only layout (removed sculpture garden/amenity dead space)
- `MapController` component auto-fits to gallery content bounds on mount and resize
- Floor switcher moved to header (always accessible)
- Smaller pin sizes (26/32px), compact popup layout with thumbnails
- `100dvh` viewport height, safe-area padding for bottom nav

## Known Issues / Notes

- **MoMA API token expired** — returns "Input or token is not valid". Artwork images sourced from GitHub CSV instead. If refreshed, can enable live search and on-view status.
- **Playwright screenshots timeout** on pages with SVG floor plans (font loading issue in Playwright MCP). App works fine in real browsers.
- **Some gallery assignments may be stale** — MoMA rotates works. The GitHub CSV `OnView` field was used where available but some artworks had no location data.
- **Teammate Justin's repo** (`github.com/justinjohnso/moma_api-explorer`) documents all MoMA API endpoints — useful reference if API access is restored.
